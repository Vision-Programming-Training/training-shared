# データモデル（DB エンティティ表）

## 1. 全体像（ER）

```
Coupon ──0..1───┐
                │ (任意)
                ▼
            ┌───────┐        ┌───────────┐        ┌─────────┐
            │ Order │1──────*│ OrderItem │*──────1│ Product │
            └───────┘        └───────────┘        └─────────┘
```

- `Order` 1 件は `OrderItem` を 1 件以上持つ（注文明細）。
- `OrderItem` は 1 つの `Product` を指す（どの商品を何個か）。
- `Order` は `Coupon` を **0 または 1 件**参照する（クーポン未使用の注文がある）。
- `Product` と `Coupon` はマスタ
- `Order` / `OrderItem` はトランザクション。

---

## 2. エンティティ表

### Product（商品マスタ）

| 列 | 論理型 | 必須 | 説明 |
|---|---|---|---|
| Id | 整数 | ○ | 主キー（自動採番） |
| Name | 文字列 | ○ | 商品名 |
| Price | 金額 | ○ | **税抜**単価 |
| Stock | 整数 | ○ | 在庫数（0 以上。在庫切れ商品は 0） |

### Order（注文）

| 列 | 論理型 | 必須 | 説明 |
|---|---|---|---|
| Id | 整数 | ○ | 主キー（自動採番） |
| Status | 列挙(OrderStatus) | ○ | 注文ステータス |
| TotalAmount | 金額 | ○ | 確定時の**税込**合計（`(税抜小計 − 割引) × 税率` を円未満四捨五入） |
| CouponId | 整数 | − | 外部キー → Coupon。クーポン未使用なら NULL |
| CreatedAt | 日時(UTC) | ○ | 注文作成日時。一覧は降順（新しい順） |

### OrderItem（注文明細）

| 列 | 論理型 | 必須 | 説明 |
|---|---|---|---|
| Id | 整数 | ○ | 主キー（自動採番） |
| OrderId | 整数 | ○ | 外部キー → Order |
| ProductId | 整数 | ○ | 外部キー → Product |
| Quantity | 整数 | ○ | 注文数量（1 以上） |
| UnitPrice | 金額 | ○ | **注文時点の税抜単価のスナップショット**（後から `Product.Price` が変わっても不変） |

> **不変条件**: 明細の単価は `Product.Price` を都度参照せず、注文確定時にコピーした `UnitPrice` を使う。これにより過去注文の金額は商品の値上げ・値下げに影響されない。表示の小計は `UnitPrice × Quantity`。

### Coupon（クーポンマスタ）

| 列 | 論理型 | 必須 | 説明 |
|---|---|---|---|
| Id | 整数 | ○ | 主キー（自動採番） |
| Code | 文字列 | ○ | クーポンコード（利用者が入力。コードで検索する） |
| DiscountType | 列挙(DiscountType) | ○ | 割引種別 |
| DiscountValue | 金額 | ○ | 割引値（`FixedAmount` なら円、`Percentage` なら％） |

> 注: `Code` は論理的には一意だが、基準実装では DB の一意制約は張っていない（seed が重複コードを入れない前提）。

---

## 3. 列挙（enum）

DB には**文字列**として保存する（読みやすさ優先。数値の意味ズレを防ぐ）。

### OrderStatus

| 値 | 意味 |
|---|---|
| Pending | 未確定（カート相当） |
| Confirmed | 確定済み |
| Cancelled | キャンセル済み |

> 基準実装では注文作成時に `Confirmed` 固定。`Pending` を作る経路は未実装（機能追加課題で扱う）。

### DiscountType

| 値 | 意味 |
|---|---|
| FixedAmount | 定額割引（円） |
| Percentage | 定率割引（％） |

---

## 4. 永続化の約束（言語版で守る点）

- **主キー**は `Id`、自動採番。
- **外部キー**: `Order.CouponId`（NULL 可）、`OrderItem.OrderId`、`OrderItem.ProductId`。
- 金額列は**固定小数点**で保存する（基準実装は `decimal(18,2)`）。浮動小数点は使わない。
- 列挙は**文字列**として保存する。
- `Order` と `OrderItem` は親子（1..*）。`Order` 削除時の挙動は基準実装の既定に従う。

---

## 5. 初期データ（seed・実値）

アプリ初回起動時に投入されるデータ。

### 商品（Product）

| Name | Price(税抜) | Stock |
|---|---|---|
| ノート | 300 | 50 |
| ボールペン | 150 | 100 |
| マグカップ | 1200 | 20 |
| Tシャツ | 2500 | 10 |
| ステッカー | 200 | 0（在庫切れ） |
| トートバッグ | 1800 | 15 |
| キーホルダー | 600 | 30 |

### クーポン（Coupon）

| Code | DiscountType | DiscountValue |
|---|---|---|
| WELCOME500 | FixedAmount | 500（円） |
| SALE10 | Percentage | 10（％） |

### 既存の注文（Order ＋ 明細）

| 注文 | Status | Coupon | 明細（商品 × 数量 @税抜単価） | TotalAmount(税込) |
|---|---|---|---|---|
| A | Confirmed | （なし） | ノート×2 @300、マグカップ×1 @1200 | 1980 ＝ (600+1200)×1.10 |
| B | Confirmed | WELCOME500 | Tシャツ×1 @2500、ボールペン×3 @150 | 2695 ＝ (2950−500)×1.10 |
| C | Confirmed | SALE10 | トートバッグ×1 @1800 | 1782 ＝ (1800−10%)×1.10 |

> 明細の `UnitPrice` は seed 時点の `Product.Price` と一致する（スナップショットの初期値）。
