# 課題A6（★★☆）存在しない注文IDを開くと「サーバーエラー」になる

セッション4: エラーの扱い（層をまたぐ）

## 現象

存在しない注文IDを指定して取得すると、「見つかりません（404）」ではなく
500（サーバー内部エラー）が返る。

## 再現手順

1. Swagger（または API 直叩き）で、存在しない注文ID（例: 存在しない大きな数値）を指定して注文取得を呼ぶ。
2. レスポンスが **500 Internal Server Error** になる。
3. 本来は **404 Not Found** が返ってほしい。

## 期待結果

- 存在しない注文IDを開いたら **404（見つかりません）** を返す。
- サーバー内部エラー（500）にはしない。

## 備考・ヒント

- `Controllers/OrdersController.cs` には try/catch が**ない**。では例外はどこで HTTP 応答に変換されているか——`Middleware/`（例外を HTTP のエラー応答に変換する場所）を読む。
- 「どの例外が」「どの HTTP ステータスに」変換されているかの**対応関係**を調べる。
- そのうえで、注文が見つからないときに**投げている例外の種類**が、404 に変換される例外と噛み合っているかを確認する。投げる側／変換する側のどちらを直すのが最小限か考える。
