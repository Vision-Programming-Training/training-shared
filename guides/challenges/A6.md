# 課題A6（★★☆）存在しない注文IDを開くと「サーバーエラー」になる

## 現象

存在しない注文IDを指定して取得すると、「見つかりません（404）」ではなく
500（サーバー内部エラー）が返る。

## 再現手順（API を直接叩く）

1. バックエンドを起動する（`http://localhost:5000` で待ち受け）。
2. 存在しない注文ID（例: 存在しない大きな数値）を指定して注文取得を呼ぶ。

   ```bash
   curl -i http://localhost:5000/api/orders/999999
   ```

   - `curl` の代わりにブラウザのアドレスバーに `http://localhost:5000/api/orders/999999` を入力してもよい。
3. レスポンスが **500 Internal Server Error** になる。
4. 本来は **404 Not Found** が返ってほしい。

## 期待結果

- 存在しない注文IDを開いたら **404（見つかりません）** を返す。
- サーバー内部エラー（500）にはしない。
