# 課題A8（★★☆）在庫を超えて注文できてしまう（条件あり）

> テストは全部緑なのにバグがある。**まず症状を再現 → それを捕まえるテストを自分で書いて赤くする → 直して緑にする** の順で。

## 現象

普通に注文する分には問題ないが、**同じ商品をカートで2行に分けて**注文すると、
在庫を超えていても注文が通ってしまうことがある。

## 再現手順

1. 在庫 = N の商品を1つ選ぶ。
2. その**同じ商品を2行**に分けてカートに入れる。
   - 例: 1行目に N 個、2行目にもう数個（合計が N を超えるようにする）。
3. その内容で注文する。
4. 合計数量が在庫 N を超えているのに、注文が通ってしまう。

## 期待結果

- 同じ商品が複数行に分かれていても、**合計数量で在庫判定**する。
- 合計が在庫を超える注文は「在庫不足」で弾く。
