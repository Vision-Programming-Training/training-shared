# 課題A1（★★☆）10%オフのクーポンがほとんど効かない

セッション1: 金額のバグを直す（読む・テストを使う）

## 現象

定率クーポン `SALE10`（10%オフ）を使っても、ほとんど割り引かれない。
一方で定額クーポン `WELCOME500`（500円引き）は正常に効く。

## 再現手順

1. 商品一覧で **トートバッグ** の「カートに追加」を押す（数量1）。
2. クーポンコード欄に `SALE10` と入力する。
3. 「この内容で注文する」を押す。
4. 表示される税込合計が、期待値 **¥1,782** から大きくずれている（ほとんど割引されていない）。

> Swagger から注文 API を直接叩いて確認してもよい。

## 期待結果

- トートバッグ1点 − `SALE10`（10%引き）= **1,782円**。
- 定率・定額のどちらのクーポンでも、ルールどおりに割引が反映される。

## 備考・ヒント

- 業務の判断は `Services/` にある。金額計算は `Services/PricingService.cs` を見る。
- 定率（`SALE10`）と定額（`WELCOME500`）の**処理の違い**に注目する。正常な定額側と比べると、定率側の計算のどこがおかしいか見えてくる。
- 割引の計算ルール: 定率は「○%引き」。引きすぎて0円未満になったら0円。
