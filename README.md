# training-shared

全言語版バックエンドで共有する、言語非依存の研修資料。

## 中身

| ファイル | 対象 | 内容 |
|---|---|---|
| `training-app-plan.md` | 教材設計者 | 教材全体の実装計画（完成形を作るフェーズの計画書） |
| `課題カタログ.md` | 講師・教材管理者 | **言語非依存**の研修課題カタログ（狙い・症状・不変条件・難度・移植時の注意）。解答情報を含むので受講者には渡さない |
| `受講者ガイド.md` | 受講者 | 課題の進め方ガイド（解答なし）。起動/テストコマンドだけ各言語で読み替える |

## 課題ドキュメントの構造（言語横断）

課題は「**言語非依存の契約**」と「**各言語の実装の都合**」で分けて管理する。

```
training-shared/課題カタログ.md          ← 言語非依存（狙い・症状・不変条件）
training-shared/受講者ガイド.md          ← 言語非依存
training-backend-csharp/docs/課題-仕込みdiff.md   ← C# 固有：実 diff・テスト名・file path
（将来）training-backend-java/docs/課題-仕込みdiff.md  ← Java 版の diff を別途起こす
```

新しい言語版バックエンドを起こすときは、カタログをそのまま流用し、各リポジトリ側に「仕込み diff 集」を新規に書く。一部の課題（A1 丸め・A8 例外→HTTP・D1 N+1 など）は言語で罠が変わるため、カタログの「移植時の注意」を必ず参照する。
